Real-Time Payments

IMPS Payment Switch

The Challenge:

Scale a real-time payment system to handle peak traffic of 600 TPS while maintaining sub-second latency and ensuring idempotency guarantees—because in payments, duplicates are not an option.

The Architecture:

Event-Driven Architecture (EDA) & CQRS Pattern

  • Write Path: Payment processing with idempotency keys and SAGA orchestration
  • Read Path: Decoupled transaction history queries using materialized views
  • Event-Driven Autoscaling: KEDA scales pods based on Kafka lag, not CPU—40% cost savings
  • Observability Pipeline: Sub-20ms alerting with OpenTelemetry distributed tracing

The Impact:

Async Payments

NEFT Payment Switch

The Challenge:

Design a resilient payment switch for high-volume asynchronous transactions with complex state management across 30+ microservices, ensuring eventual consistency and automated failure recovery.

The Architecture:

SAGA Pattern with Temporal Workflows

  • Choreography-based SAGA: Event-driven compensation for failed transactions
  • Temporal Workflows: Durable execution for long-running payment processes
  • Idempotency Layer: Preventing duplicate payments with distributed locks
  • Automated Security: AI-powered vulnerability scanning in CI/CD pipeline
